The Honest Core

The value accrual problem.

Does document tokenization create demand for the token? The framework's hardest question, placed early on purpose.

Conditional analysis

This series argues from stated assumptions. The purpose is not to promise outcomes, but to show what the math and structure would require if adoption conditions are met — and to price the probability that they are met at all.

This part has not yet been published. The series contents are visible from day one by design; each part is written in dedicated sessions and published here when it meets the framework's sourcing standard.

What this part will cover

  • The three demand channels for the native token: gas, masternode staking collateral, and emergent protocol collateral — each verified against primary network documentation.
  • The USDC settlement problem: settlement value flowing through Circle's rails rather than the native token.
  • The inverted-RLUSD structure: where the XRP framework argued a stablecoin cannot substitute for a bridge asset at the seam, here we must test whether the stablecoin does substitute for the native token in the settlement role.
  • TradeTrust chain-agnosticism as a standing threat.
  • Supply overhang: circulating versus total supply, verified on-chain.
  • The elasticity question, derived formally. Part IV quantifies it.

Primary sources

  • XDC Network — technical documentation (masternode and staking parameters)
  • XDCScan — on-chain supply and network data
  • Circle — USDC documentation
  • IMDA (Singapore) — TradeTrust documentation
